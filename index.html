🐙التخزين السحابي — GitHub (مُوصى به)

احفظ التقارير في مستودع GitHub وشاركها بروابط قصيرة جداً تعمل فوراً بدون أي انتظار.

📖 دليل الإعداد خطوة بخطوة (اضغط للتوسيع)

١. إنشاء مستودع GitHub

  1. سجّل دخولك في github.com
  2. اضغط + → New repository
  3. سمّه مثلاً rasd-daily واجعله Public
  4. فعّل Add a README file ثم اضغط Create repository

٢. تفعيل GitHub Pages

  1. من إعدادات المستودع → Pages
  2. Source: Deploy from a branch
  3. Branch: main → مجلد / (root)
  4. اضغط Save
  5. الرابط سيكون: https://username.github.io/rasd-daily/

٣. رفع ملف index.html

  1. من صفحة المستودع اضغط Add file → Upload files
  2. ارفع ملف index.html هذا
  3. اضغط Commit changes

٤. إنشاء رمز وصول (Token)

🔗 الطريقة الأسهل: اضغط هنا لإنشاء التوكن تلقائياً بالإعدادات الصحيحة — ستحتاج فقط لاختيار المستودع ومدة الصلاحية ثم الضغط على Generate token.

أو يدوياً:

  1. اذهب إلى Settings → Developer settings → Fine-grained tokens → Generate new token
  2. Token name: rasd-daily
  3. Expiration: اختر المدة المناسبة
  4. Repository access: اختر Only select repositories ثم اختر مستودع rasd-daily
  5. انزل للأسفل إلى قسم Permissions
  6. اضغط على Repository permissions لتوسيع القائمة (ستظهر قائمة طويلة من الصلاحيات)
  7. ابحث عن Contents في القائمة واضغط على القائمة المنسدلة بجانبها وغيّرها من No access إلى Read and write
  8. ملاحظة: عند اختيار Contents سيُضاف تلقائياً Metadata: Read-only — هذا طبيعي
  9. اضغط Generate token في أسفل الصفحة وانسخ الرمز فوراً (لن يظهر مرة أخرى)

⚠️ بديل أبسط: إذا لم تجد الخيارات أعلاه، يمكنك استخدام Classic Token بصلاحية repo — أسهل لكن أقل أماناً (يمنح وصولاً لكل مستودعاتك).

٥. أدخل البيانات أدناه

ألصق الرمز واسم المستخدم واسم المستودع في الحقول ثم احفظ.

بعد الإعداد: عند نسخ رابط التقرير سيكون قصيراً مثل:
https://username.github.io/rasd-daily/?date=2026-04-19

☁️التخزين السحابي — جوجل درايف (بديل)

خيار بديل يتطلب إعداد مشروع في Google Cloud.

📖 دليل الإعداد الكامل (اضغط للتوسيع)

١. إنشاء مشروع في Google Cloud

  1. ادخل إلى console.cloud.google.com
  2. أنشئ مشروعاً جديداً باسم rasd-daily

٢. تفعيل Google Drive API

  1. APIs & Services → Library → ابحث عن Google Drive API → Enable

٣. OAuth Consent Screen

  1. User Type: External → أضف بريدك كـ Test user

٤. إنشاء OAuth Client ID

  1. Credentials → Create → OAuth client ID → Web application
  2. أضف رابط موقعك في Authorized JavaScript origins
  3. انسخ Client ID

٥. إنشاء API Key

  1. Credentials → Create → API key → قيّده بـ Google Drive API فقط
